# Analysis Skill — 評測範例

## 好的輸出範例

> 題目：評估一人公司是否應在 2026 年進入越南市場（Go/No-Go 決策）

```markdown
# 越南市場進入決策（2026）

## 結論與建議
建議：**分階段試探，不全面進入**。
先以 3 個月 Discovery 專案測試市場水溫，目標找 1-2 個錨點客戶，
驗證付費意願後再決定是否全力投入。直接 No-Go 風險是錯過早期先行者優勢；
直接全面進入風險是現金流斷裂（一人公司無法承受 6+ 個月無收入燒錢期）。

## 選項比較

### 選項 A：立即全面進入
| 維度 | 評估 | 依據 |
|------|------|------|
| 價值 | 高（潛在市場 NT$2-5M/年） | 越南 GDP 成長 6.5%，AI 顧問需求上升（World Bank 2025） |
| 成本 | 高（NT$500K+ 前期投入） | 簽證、法律、人脈建立，估 3 個月全力投入 |
| 風險 | 高 | 無本地人脈、語言障礙、現金流風險 |
| 可行性 | 中 | 技術上可行，執行難度大 |
| 執行難度 | 高 | 需同時維持台灣業務 |
| 預期回報 | 12 個月後 NT$1M+（高風險） | 假設：3 個錨點客戶，每家 NT$300K/年 |
| 一人公司適配度 | 低 | 超出單人可執行邊界 |

### 選項 B：分階段試探（建議）
| 維度 | 評估 | 依據 |
|------|------|------|
| 價值 | 中高（驗證成本低） | 3 個月花 NT$50K 驗證，失敗損失可控 |
| 成本 | 低（NT$30-50K 差旅+時間） | 1-2 次出差，線上 Discovery call 為主 |
| 風險 | 低-中 | 失敗最多損失 NT$50K 和 3 個月時間 |
| 可行性 | 高 | 可在台灣業務間隙執行 |
| 執行難度 | 中 | 需要越南人脈引薦，但不需全職投入 |
| 預期回報 | 3 個月驗證期後決定是否加碼 | 試探成本低，機會保留 |
| 一人公司適配度 | 高 | 控制在可執行範圍內 |

### 選項 C：維持現狀（不進入）
| 維度 | 評估 | 依據 |
|------|------|------|
| 價值 | 無新增 | 維持台灣現有業務 |
| 成本 | 零 | 不需額外投入 |
| 風險 | 低（但有機會成本） | 錯過越南早期市場窗口 |
| 可行性 | 高 | 維持現狀最容易 |
| 執行難度 | 低 | |
| 預期回報 | 零增長 | |
| 一人公司適配度 | 高（但不成長） | |

## 高風險假設
- **「越南客戶付費意願等同台灣」**：若不成立，整個商業模式需重算
  （越南 IT 顧問日費中位數約台灣的 40-60%）
- **「3 個月能找到錨點客戶」**：若不成立，建議在第 8 週重評是否繼續

## 待驗證
- 越南本地 AI 顧問競爭者數量與定價（驗證方式：LinkedIn 搜尋 + 詢問引薦人）
- 目標 ICP（中型製造業/科技業）的決策週期（驗證方式：第一次 Discovery call）

## 建議下一步
1. **本週**：透過越南朋友確認是否有引薦機會（成本：0）
2. **第 2 週**：若有引薦，安排一次線上 Discovery call（成本：2 小時）
3. **第 4 週**：依據 call 結果決定是否安排實地出差
```

**為什麼好**：結論先行且明確說明理由；每個選項都有評估（包含「不做」）；
成本和風險具體量化（NT$金額、時間）；高風險假設明確標出；下一步可執行。

## 壞的輸出範例

> 同一題目的錯誤輸出

```markdown
# 越南市場分析

越南是東南亞成長最快的市場之一，GDP 年成長率高達 7%，人口年輕化，
數位化程度快速提升。對於希望拓展海外市場的企業而言，越南提供了
絕佳的機會。

## 優缺點

**優點**
- 市場大
- 成長快
- 競爭少

**缺點**
- 語言障礙
- 文化差異
- 法規複雜

## 結論
綜合以上分析，進入越南市場有其機會與挑戰。建議公司根據自身條件
審慎評估後做出決定。
```

**哪裡錯**：
- 結論無效（「審慎評估後做出決定」不是決策建議，是推卸責任）
- 沒有選項比較，無法支持 Go/No-Go 判斷
- 「競爭少」無依據，屬高風險假設卻未標記
- 優缺點只有形容詞，沒有數字和來源
- 完全沒有考慮一人公司的資源限制
- 沒有下一步行動，結論無法執行

## 判斷標準

| 項目 | 通過 | 不通過 |
|------|------|--------|
| 結論 | 有明確建議選項及理由 | 「審慎評估」「視情況而定」等模糊語句 |
| 完整性 | 所有選項都有評估，含「不做」 | 只推薦一個選項，其他略過 |
| 量化 | 成本/風險有具體金額或時間 | 只說「高」「低」無數字 |
| 假設 | 高風險假設明確列出並說明影響 | 把假設當事實寫入分析 |
| 執行性 | 下一步具體可操作（誰做、何時、成本多少） | 沒有下一步或下一步太抽象 |
| 一人公司視角 | 每個選項評估執行可行性 | 忽略資源限制，假設有無限人力 |
