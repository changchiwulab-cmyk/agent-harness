# Analysis Skill — 評測範例

## 好的輸出範例

> 題目：評估一人公司是否應導入付費 AI 寫作工具（月費 NT$600）

```markdown
# 是否導入付費 AI 寫作工具

## 結論與建議
**建議：導入（選項 A）**。月費 NT$600 相當於 0.6 小時人力成本（以時薪 NT$1,000 估算），
但預期每週可節省 3-5 小時的草稿與校稿時間。回收期約 2 週。唯一風險是工具學習曲線，
建議先試用 14 天免費版確認工作流程相容後再付費。

## 選項比較

### 選項 A：導入付費 AI 寫作工具（月費 NT$600）
| 維度 | 評估 | 依據 |
|------|------|------|
| 價值 | 高 | 每週省 3-5 小時草稿時間（合理推論：基於同類工具使用者報告） |
| 成本 | 低 | NT$600/月，約 0.6 小時人力成本 |
| 風險 | 低 | 工具鎖定風險低（可隨時取消），資料隱私需確認條款 |
| 可行性 | 高 | 14 天免費試用可驗證，無硬體需求 |
| 執行難度 | 低 | 學習曲線約 2-4 小時（已知事實：官方文件標示） |
| 預期回報 | 高 | 回收期 ≤ 2 週（NT$600 ÷ NT$1,000/hr ÷ 3hr/週省時 ≈ 0.2 週） |
| 一人公司適配度 | 高 | 固定月費可預測，省時效果對一人公司邊際效益大 |

### 選項 B：使用免費版（功能受限）
| 維度 | 評估 | 依據 |
|------|------|------|
| 價值 | 中 | 免費版每日限額 10 次，不足以涵蓋每日寫作需求 |
| 成本 | 零 | 無直接費用 |
| 風險 | 低 | 無財務風險，但效率瓶頸持續 |
| 可行性 | 高 | 立即可用 |
| 執行難度 | 低 | 同上 |
| 預期回報 | 低 | 限額制約導致只能處理部分任務，省時效果打折 |
| 一人公司適配度 | 低 | 限額在高產量週會成為瓶頸 |

### 選項 C：不做（維持現有工作流程）
| 維度 | 評估 | 依據 |
|------|------|------|
| 價值 | 零增量 | 維持現狀無新增效益 |
| 成本 | 每週 3-5 小時隱性成本 | 合理推論：現有草稿時間評估 |
| 風險 | 低 | 無新風險，但機會成本持續 |
| 可行性 | 高 | 無需任何改變 |
| 執行難度 | 零 | 維持現狀 |
| 預期回報 | 負（機會成本）| 競爭者若採用工具，效率差距拉大 |
| 一人公司適配度 | 低 | 一人公司時間是最稀缺資源，放棄省時工具代價高 |

## 高風險假設
- **「每週省 3-5 小時」**：若實際省時不足 1 小時，ROI 轉負。驗證方式：14 天試用期追蹤實際使用頻率與時間節省。
- **「資料不會用於訓練」**：需確認服務條款，若客戶資料需保密則需重新評估。

## 待驗證
- 工具是否支援繁體中文輸出品質（試用期驗證）
- 與現有工作流程（Notion/Google Docs）整合難度

## 建議下一步
1. 啟動 14 天免費試用（今天）
2. 試用第 7 天：評估省時效果是否達到每週 2 小時門檻
3. 第 14 天決策點：達標則付費，未達標則選選項 B
```

**為什麼好**：
- 結論與建議在最前面，含具體行動
- 每個評估有具體數字或可追溯依據
- 三個選項全部評估，包含「不做」（選項 C）
- 明確標示已知事實 vs 合理推論
- 高風險假設有驗證方法，不只列出
- 一人公司適配度每個選項都有評估

## 壞的輸出範例

> 同一題目的錯誤輸出

```markdown
# AI 寫作工具評估

## 優點
- 節省時間
- 提升寫作品質
- 現代化工作流程

## 缺點
- 需要費用
- 有學習成本
- 可能不準確

## 結論
AI 工具有很多優點，但也有缺點。如果預算允許，可以考慮使用。
建議先試試看，視情況決定。
```

**哪裡錯**：
- 只有優缺點列表，沒有選項比較（選項 B 和「不做」完全消失）
- 完全沒有六維評估結構
- 沒有任何具體數字（「節省時間」節省多少？）
- 結論模糊不可執行（「視情況決定」是什麼情況？）
- 沒有區分已知事實 / 合理推論 / 高風險假設
- 缺少「建議下一步」
- 完全沒評估一人公司適配度

---

## 判斷標準

| 項目 | 通過 | 不通過 |
|------|------|--------|
| 選項完整性 | 所有選項含「不做」都評估 | 只推薦一個選項，其他忽略 |
| 六維評估 | 每個選項都有完整表格 | 只列優缺點 |
| 具體數字 | 成本、時間、回報有數值 | 「成本高」「省時多」等模糊詞 |
| 結論可執行 | 明確建議 + 下一步行動 | 「視情況」「可以考慮」 |
| 假設透明 | 高風險假設有標示與驗證方式 | 把推論當事實直接寫入結論 |
| 一人公司適配 | 每個選項都評估資源可行性 | 忽略執行者只有一人的限制 |
