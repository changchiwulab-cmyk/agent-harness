# Analysis Skill

## 用途
決策支援、方案評估、Go/No-Go 判斷、策略分析、可行性評估。

與 research 的差異：research 回答「事實是什麼」，analysis 回答「該怎麼選」。

## 執行流程
1. 確認決策問題（從 Task Card 的 goal 取得）
2. 列出決策選項（包含「不做」選項）
3. 對每個選項跑六維評估：價值、成本、風險、可行性、執行難度、回報
4. 明確區分：已知事實 / 合理推論 / 待驗證 / 高風險假設
5. 加一層判斷：是否適合一人公司採用
6. 給出建議排序與理由
7. 輸出到指定位置

## 輸出格式
```markdown
# [決策主題]

## 結論與建議
[一段話：建議選哪個，為什麼]

## 選項比較

### 選項 A：[名稱]
| 維度 | 評估 | 依據 |
|------|------|------|
| 價值 | | |
| 成本 | | |
| 風險 | | |
| 可行性 | | |
| 執行難度 | | |
| 預期回報 | | |
| 一人公司適配度 | | |

### 選項 B：...

## 高風險假設
- [假設1]：如果不成立，結論會怎麼變

## 待驗證
- [項目1]：驗證方式

## 建議下一步
1. ...
```

## 品質標準
- 每個選項都要評估，不能只推薦一個而忽略其他
- 「不做」永遠是一個選項
- 成本和風險要具體化（金額、時間、機率），不要只說「高/低」
- 結論必須可追溯到分析內容，不能憑空出現

## 常見失敗模式
- 只列優缺點不給建議 → 必須有明確的排序和建議
- 遺漏「不做」選項 → 維持現狀永遠要評估
- 用模糊詞代替具體評估（「成本較高」→ 高多少？）→ 盡量量化
- 忽略一人公司的資源限制 → 每個選項都要評估執行可行性
