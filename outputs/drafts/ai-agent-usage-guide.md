# AI Agent 使用說明

## 前言

本 AI Agent 是一套設計給一人公司或小型顧問團隊使用的任務執行助理。它不是全自動機器人，而是一位需要你給出明確指令才會行動的高效執行夥伴。核心設計原則只有一個：**可控勝於全能**。每個任務都有書面紀錄，每個輸出都先到草稿區等你確認，不會有任何意外動作。

---

## 快速開始

執行一項任務需要三個步驟：

**步驟一：建立任務卡（Task Card）**

複製 `tasks/TASK_CARD_TEMPLATE.yaml`，建立一個新檔案（命名格式：`YYYY-MM-DD_任務簡述.yaml`）。填入以下必要欄位：
- `goal`：一句話說明要做什麼
- `definition_of_done`：列出 2-5 條可驗證的完成標準
- `skill_type`：選擇任務類型（research / writing / analysis / review / ops）

其餘欄位有預設值，初次使用可直接沿用。

**步驟二：告訴 Agent 執行**

在 Claude Code CLI 中輸入：「執行 tasks/你的任務檔案.yaml」。Agent 會先確認目標和完成標準，確認無誤後開始作業。

**步驟三：確認輸出**

Agent 完成後，輸出草稿會儲存在 `outputs/drafts/` 資料夾中。你審閱後決定是否採用，Agent 不會自動對外發送任何內容。

---

## 常見錯誤

**錯誤一：沒有填寫 `definition_of_done`**

這是最常見的問題。空白的完成標準會讓 Agent 不知道何時停止，導致輸出品質不穩定。解法：每個任務至少寫 2 條可驗證的完成條件，例如「輸出超過 500 字」、「涵蓋至少 3 個選項」。

**錯誤二：要求跳過草稿直接執行對外動作**

Agent 設計上不支援直接發送 Email、更新正式資料或公開發文。所有對外動作都需經過草稿階段，由你人工確認後再執行。這是安全機制，不是 bug。

**錯誤三：中途改變任務目標**

如果執行到一半你想改變方向，不要在對話中直接說「改成 XX 方向」。正確做法是先請 Agent 暫停，更新 Task Card 的 `goal` 欄位，再繼續執行。這樣才能保留完整的決策紀錄。

---

## 注意事項

**關於輸出位置**：所有輸出一律進 `outputs/drafts/`，未經確認不會進入 `outputs/reports/`（正式版）。草稿只代表「Agent 認為任務完成」，不代表「可以直接使用」。

**關於失敗處理**：如果 Agent 連續失敗 3 次，它會自動停下來並記錄錯誤，等待你的指示。不要期望它自行解決所有問題——設計上就是要讓人介入，這比靜默失敗安全得多。

**關於 Token 成本**：每次對話都有成本。長任務建議拆成小任務分批執行，而不是一次塞入大量需求。能用本機檔案讀取解決的，不要觸發網路搜尋。

**關於版本記錄**：每個關鍵節點 Agent 會自動建立 git commit（稱為 checkpoint），方便你事後追蹤任何輸出是在哪個階段產生的。不需要手動管理版本。
