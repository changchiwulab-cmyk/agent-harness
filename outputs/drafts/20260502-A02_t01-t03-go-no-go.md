## 選項比較

### 選項 A：重產四張卡全鏈，最終升級 reports/

| 維度 | 評估 | 依據 |
|---|---|---|
| 價值 | 中–高 | 若有使用場景，得到完整可用的策略提案 v2 |
| 成本 | 高 | ~30-40K tokens + 3-4 hr 人工 review + 3 輪 web_search 配額（S01）|
| 風險 | 中 | 重產結果不會等於原 v2（web_search 時間性）；可能再經一輪 RV→修正循環 |
| 可行性 | 高 | skills/research/writing/review/ops 全部就位，pipeline 已驗證可跑 |
| 執行難度 | 中 | 4 張卡序列依賴，中間任一階段失敗要 rollback |
| 預期回報 | 條件性 | **高（若有客戶要用）/ 低（若只是練習）** |
| 一人公司適配度 | 中 | 會吃掉 0.5-1 個工作天，排擠其他主動工作 |

### 選項 B：只重產 v2 等價物，跳過 v1+review

| 維度 | 評估 | 依據 |
|---|---|---|
| 價值 | 中 | 直接得到「已修正版」的策略文件，省一輪審查 |
| 成本 | 中 | ~15-20K tokens + 1.5-2 hr，無需 web_search（context 用既有 audit summary）|
| 風險 | 中–高 | 跳過 RV 等於跳過品質 gate；數字矛盾類錯誤可能再現 |
| 可行性 | 中 | 需要新開一張 writing 卡，DoD 要併入 RV01 三項必修，schema 上做得到 |
| 執行難度 | 中 | 比 A 簡單，但失去獨立審查的客觀性 |
| 預期回報 | 中 | 比 A 快得到結果，但品質保證較弱 |
| 一人公司適配度 | 中–高 | 比 A 省一半時間 |

### 選項 C：全部封存到 tasks/archived/，不留 memo

| 維度 | 評估 | 依據 |
|---|---|---|
| 價值 | 低 | 認賠出場，現在 review 隊列乾淨 |
| 成本 | 極低 | ~10 min，移檔 + 改 status: archived |
| 風險 | 中 | 4 週前的研究結論（5 商業模式、台越市場分析）完全沒留下任何可追溯線索 |
| 可行性 | 高 | 純檔案操作 |
| 執行難度 | 低 | — |
| 預期回報 | 負 | 之後若想再用，要從零開始 |
| 一人公司適配度 | 低 | 違反 GLOBAL_RULES「保留可追溯性」精神 |

### 選項 D：on_hold + 寫 1-page memo（**首選**）

| 維度 | 評估 | 依據 |
|---|---|---|
| 價值 | 中 | 保留結論層（5 商業模式、4 服務、台越 ICP、3+6 修正項摘要），可追溯 |
| 成本 | 低 | ~5-10K tokens + 30-45 min（從 audit log 摘要 + Task Card context 編寫 memo）|
| 風險 | 低 | 完全可逆；未來若要重產，memo 本身就是新一輪 W01 的高品質 input |
| 可行性 | 高 | 純內部檔案操作；屬 ops + writing 範圍 |
| 執行難度 | 低 | 單一文件，不需要序列依賴 |
| 預期回報 | 高（風險調整後）| 用最小投入鎖住 4 週前的決策結論 |
